计算机网络期末复习

1. 概述

一、概述

  1. 互联网的拓扑结构从组成方式上可以分为：

    • 边缘部分：用户直接使用的，用于进行通讯和资源共享。
    • 核心部分：为边缘部分提供服务的。核心部分起特殊作用的是路由器，路由器是实现分组交换的关键组件，它的任务是转发收到的分组。

  2. internet互联网能向用户提供的服务：连通性和共享。

  3. 因特网提供的网络服务有哪些

    答：远程登陆服务（Telnet），文件传输服务（FTP），电子邮件服务（E-mail），网络新闻服务（Usenet），名址服务(Finger、Whois、X.500、Netfind)，文档查询索引服务(Archie、WAIS)，信息浏览服务(Gopher、WWW)，其他信息服务(Talk、IRC、MUD)。

  4. 计算机网络是计算机技术和通信技术相结合的产物。（✅）

  5. 网络边缘的端系统的通讯方式有以下两种：

    • 客户-服务器方式（C/S）：特征是客户是请求方，服务器是提供方，客户和服务器都使用核心部分提供的服务。客户和服务器指的是计算机进程。

    • 对等方式（P2P）：两台主机在进行通讯时不区分服务提供方和服务请求方。

  6. 计算机网络的类别

    按作用范围划分：

    • 广域网（WAN，Wide Area Network）：几十到几千km
    • 城域网（MAN，Metropolitan Area Network）：5~50km
    • 局域网（LAN，Local Area Network）：1km
    • 个人区域网（PAN，Personal Area Network）

    按照使用者分类：

    • 公用网（public network）
    • 专用网（private network）

  7. OSI、TCP/IP

    开放系统互联基本参考模型OSI（Open System Interconnect Reference Model）

  8. TCP/IP模型包含的层次及功能

    • 网络接口层：这是TCP/IP模型的最低层，负责接收从IP层交来的IP数据报并将IP数据报通过低层物理网络发送出去，或者从低层物理网络上接收物理帧，抽出IP数据报，交给IP层。

    • 网络层：负责为分组交换网上的不同主机提供通讯服务

    • 运输层：其任务是负责向两台主机中的进程之间的通讯提供通用的数据传输服务。
    • 应用层：其任务是通过应用进程之间的交互完成特定的网络应用。

  9. 网络协议：为进行网络中的数据交换而建立的规则、标准或规定称之为网络协议。

    网络协议的三个要素：

    • 语法
    • 语义
    • 同步

    实体是任何可接受或发送信息的硬件或软件进程。

    协议是控制两个对等实体（或多个实体）进行通讯的规则的集合。

  10. 协议和服务的区别：

    协议的实现保证了能够向上一层提供服务。本层的服务用户只能看见服务而无法看见下面的协议。即下面的协议对上面的服务用户是透明的。

    协议是“水平的”，即协议是控制两个对等实体进行通信的规则。但服务是“垂直的”，即服务是由下层通过层间接口向上层提供的。上层使用所提供的服务必须与下层交换一些命令，这些命令在OSI中称为服务原语。

  11. ATM网络的“打包”最大特点是“小步快跑”的快速分组交换方式。（✅）

  12. Linux和Unix适合作用网络服务器的基本平台么？（✅）

  15. 无线局域网不使用CSMA/CD，而使用CSMA/CA的主要原因是_C_。

    A．不需要在发送过程中进行冲突检测
    B．无线信号的广播特性，使得不会出现冲突
    C．不能同时收发，无法在发送时接收信号
    D．覆盖范围较小，不进行冲突检测，不影响正确性

    CSMA/CA是IEEE 802.11无线局域网(WLAN)的MAC子层协议，主要用于解决无线局域网的信道共享访问问题。而在采用IEEE 802.3标准的以太网中，MAC子层采用CSMA/CD协议。这两种协议都针对网络中共享信道如何分配的问题，但它们的工作原理却有所不同。最明显的区别是：CSMA/CA是在冲突发生前进行冲突处理，而CSMA/CD是在冲突发生后进行冲突处理。导致这种不同的根本原因在于，WLAN所采用的传输媒介和传统LAN所采用的传输媒介有着本质的区别。也正是这种区别，导致WLAN存在新的问题：隐藏站问题和暴露站问题。这些问题都属于隐蔽终端问题。对于采用CSMA/CA协议的无线局域网络(WLAN)，使用了预约信道、ACK帧回避机制、RTS/CTS回避机制来实现冲突避免。无论是ACK帧回避机制还是RTS/CTS回避机制，都因为增加了额外的网络流量，所以在网络利用率方面，IEEE802.11无线网络与类似的以太网相比，性能总是差一点。

    从另一个角度分析，在无线局域网中实现冲突检测比较困难，其主要原因在于：①要检查是否存在冲突，需要无线连接设备一边传送数据，一边接收数据，这对于无线设备而言比较难以实现；②无线介质上的信号强度动态的变化范围很广，因此发送方无法使用信号强度的变化来判断是否出现了冲突；③在无线通信中存在“隐藏站”问题等。

  16. 网络上软硬件共享资源包括 服务器 、 打印机 、 文件 、 工具软件与应用软件 和 数据 。

物理层

  1. 串行/并行传输（没有绝对）

    串行传输：数据在一个信道上按位的次序传输的方式

    并行传输：数据在多个信道上同时传输的方式。

    数据在计算机内部一般采用并行传输，但是数据在通信线路中一般是串行传输，这是出于经济的考虑。

  2. 从通讯双方的交互方式来看，有三种基本方式：

    • 单向通讯（单工通讯）：只能有一个方向上的通讯，如无线电广播、有线电广播、电视信号
    • 双向交替通讯（半双工通讯）：双方都可以发送/接受信息，但是不能同时发送/接受。
    • 双向同时通讯（全双工通讯）：双方可以同时发送/接受信息。

  3. 信道复用技术，即共享信道进行通讯。

    • 频分复用FDM：所有用户在同一时间占用不同的带宽资源（频率带宽）。
    • 时分复用TDM：所有用户在不同时间占用相同的频带宽度。
    • 统计时分复用STDM：一种改进的时分复用。
    • 码分复用CDM：每一个用户在相同的时间内使用相同的频带进行通讯，各用户使用了特殊挑选的不同码型，所以不会有干扰。
    • 波分复用WDM：光的频分复用。

  4. 外置式调制解调器，一般应该接在计算机的串行口上。

  5. 光纤通信的优缺点：

    光纤通信的优点：

    （1）容许频带很宽，传输容量很大

    （2）损耗低、中继距离长

    （3）保密性能好

    （4）抗电磁干扰性能强

    （5）节省有色金属和原材料

    （6）体积小、重量轻

    光纤通信的缺点：

    （1）抗拉强度低

    （2）光纤连接困难

    （3）光纤怕水

  6. 物理层的接口有哪几个方面的特性？各包含些什么内容？

    （1）机械特性:说明接口所用的接线器的形状和尺寸、引线数目和排列、固定和锁定装置等等。

    （2）电气特性:指明在接口电缆的各条线上出现的电压的范围。

    （3）功能特性:指明某条线上出现的某一电平的电压表示何意。

    （4）规程特性:说明对于不同功能的各种可能事件的出现顺序。

数据链路层

  1. 要发送的数据为1101011011。采用CRC(循环冗余检验)的生成多项式是P(X)=X^4^+X+1。试求应添加在数据后面的余数。数据在传输过程中最后一个1变成了0，问接收端能否发现？若数据在传输过程中最后两个1都变成了0，问接收端能否发现？采用CRC检验后，数据链路层的传输是否就变成了可靠的传输？

    p=10011,n=4,m=1101011011,k=10

    1101011011000/10011商为110000101，余数为1100，所以添加的检验序列（帧检验序列FCS）是 1110。

    两种错误均可发现。

    因为CRC校验只能保证无比特差错出现，但仍可能出现传输差错（帧丢失、帧重复、帧失序），而且缺少重传机制，数据链路层的传输还不是可靠的传输。

  2. 网络适配器的作用是什么? 网络适配器工作在哪一层?

    适配器（即网卡）来实现数据链路层和物理层这两层的协议的硬件和软件网络

    适配器工作在TCP/IP协议中的网络接口层。

  3. 数据链路层的三个基本问题(封装成帧、透明传输和差错检测)为什么都必须加以解决？

    答：封装成帧是分组交换的必然要求，透明传输避免消息符号与帧定界符号相混淆，差错检测防止差错的无效数据帧浪费后续路由上的传输和处理资源

  4. 有固定基础设施的无线局域网中的一个站检测到信道处于忙态，或者是由于物理层的载波监听检测到信道忙，或者是由于MAC层的虚拟载波监听机制指出了信道忙。

  5. 试说明 10BASE-T各数字或字母代表的含义：

    10代表10Mbps的数据传输率，BASE表示连接线上的信号是基带信号，T代表双绞线。

  6. 网桥的工作原理和特点是什么？网桥与转发器以及以太网交换机有何异同？

    网桥工作在数据链路层，它根据MAC帧的目的地址对收到的帧进行转发。网桥具有过滤帧的功能。当网桥收到一个帧时，并不是向所有的接口转发此帧，而是先检查此帧的目的MAC地址，然后再确定将该帧转发到哪一个接口。转发器工作在物理层，它仅简单地转发信号，没有过滤能力。以太网交换机则为数据链路层设备，可视为多端口网桥。

  7. 试比较网络互连设备网桥、路由器、网络协议变换器在对应协议层功能方面的主要区别。

    A.网桥用于数据链路层以上相同的网络，用于连接多个网段和多个服务器网际过滤。

    B.路由器用于网络层以上相同的网络，对转发路径进行选择。

    C.各层均不相同的网络，对不同类型网络协议进行转换。

  8. 网桥、交换机、集线器、转发器

    网桥：网桥属于二层设备，像是一个聪明的中继器（Repeater），除了可以扩展网络的物理连接范围外，还可以对MAC 地址进行分区，隔离不同物理网段之间的碰撞（也就是隔离“冲突域”），网桥一般只有两个端口。

    集线器是工作在物理层的设备，首先是节点发信号到线路，集线器接收该信号，因信号在电缆传输中有衰减，集线器接收信号后将衰减的信号整形放大，最后集线器将放大的信号广播转发给其他所有端口。

    交换机与网桥的工作原理类似，它也是一个二层设备，不过它支持多个端口，并且可以逻辑上将不同端口进行分组，得到所谓的VLAN。此外，交换机的数据传输速率比网桥快。

    路由器是一个三层设备，用来连接不同IP网段的各子网。

  9. 局域网将数据链路层分割为哪两个子层？这两个子层分别完成了什么功能？

    局域网将数据链路层划分为两个子层：逻辑链路控制LLC子层和介质访问控制MAC子层，从而使LAN体系结构能适应多种传输介质。因此，对各种类型的局域网来说，其物理和MAC子层需要随着所采用介质和访问方法的不同发生改变，而这些不同对于LLC子层来说都是透明的。

    （1）LLC作为数据链路层的一个子层，使用MAC子层为其提供的服务，通过与对等实体LLC子层的交互为它的上层网络层提供服务。

    （2）MAC子层是用来实现介质访问控制的网络实体。MAC子层主要功能包括数据帧的封装/拆封、帧的寻址与识别、帧的接收与发送、链路的管理、帧的差错控制及MAC协议的维护等。

  10. 简要说明电路交换和存储器转发交换这两种交换方式，并加以比较。

    电路交换是一种直接的交换方式，它为一对需要进行通信的装置之间提供一条临时的专用通道，即提供一条专用的传输通道，即可是物理通道又可是逻辑通道。这条通道时由节点内部电路对节点间传输路径通过适当选择、连接而完成的，由多个节点和多条节点间传输路径组成的链路，例如，目前公用电话网广泛使用的交换方式是电路交换。

    存储转发交换方式又可以分为报文存储转发方式与报文分组存储转发交换，报文分组存储转发交换方式又可以分为数据包与虚电路方式。分组交换属于“存储转发”交换方式，但它不像报文交换那样以报文为单位进行交换、传输，而是以更短的、标准的“报文分组”为单位进行交换传输。分组经过通信网络到达终点有两种方法：虚电路和数据报。

    与电路交换相比，报文交换方式不要求交换网为通信双方预先简历条专用的数据桐庐，因此不存在简历电路和拆除电路的过程。

  7. 以太网帧为什么要规定最小帧长？

    在以前的以太网络中，网络中的主机是以总线形式连接在一起的，为了防止信号域的冲突，采用带有冲突检测的载波监听技术（CSMA/CD）可以达到多用户共享信道的目的，而在采用CSMA/CD技术时，为确保最远的主机可以及时接收到冲突信号，就必须要求主机发送完数据帧前必须可以接送到冲突信号，若帧长太短，就无法达到该效果，从而错误地认为没有发生冲突。在以前的以太网中，争用期为51.2us，带宽为10Mbps，因而最小帧长为512b即64字节。

  8. 使用TCP对实时话音数据的传输有没有什么问题？使用UDP在传送数据文件时会有什么问题？

    答：如果语音数据不是实时播放（边接受边播放）就可以使用TCP，因为TCP传输可靠。接收端用TCP讲话音数据接受完毕后，可以在以后的任何时间进行播放。但假定是实时传输，则必须使用UDP。 UDP不保证可靠交付，但UCP比TCP的开销要小很多。因此只要应用程序接受这样的服务质量就可以使用UDP。

  1. 信带宽为1Gbps，端到端传播延时10ms，TCP的发送端窗口为65535字节，试问，可能达到的最大带宽是多少？信道的利用率是多少？

    报文的传输时延：TS=65535×8/(1×10^9^b/s)=0.524ms

    往返时延等于两倍的端到端传播时延，传输一个TCP报文段的时间为 20ms+0.524ms=20.524ms （确认信息很小不考虑发送时延）

    最大吞吐量为一个窗口的大小除以一个传输轮次的时间， 65535×8/20.524ms=25.5Mb/s。

    信道利用率= 0.524/20.524=2.55%

  2. 计算题：一个TCP连接下面采用256kbps的链路，其端到端延时为128ms，经测试，发现吞吐量只有120kbps，试问滑动窗口W是多少？（两种答案，最大和最小）

    设滑动窗口为W（bit），传输W的用时为T（T=W/256）。

    （1）若接受端在受到窗口数据的最后一个数据后，才发送确认，则发送端经过(T+128*2)时间后才能发送下一个窗口，在此条件下：

    W/（W/256+128*2）= 120

    解得：

    W≈57826

    （2）若接受端每收到一个很小的报文段后就发送确认，极端情况下，发送端每经过128*2时间就发送下一个窗口，在此条件下：

    W/（128*2）= 120

    解得：

    W=30720

  3. 试说明运输层在协议栈中的地位和作用，运输层的通信和网络层的通信有什么重要区别？为什么运输层是必不可少的？

    运输层处于面向通信部分的最高层，同时也是用户功能中的最低层，向它上面的应用层提供服务。运输层为应用进程之间提供端到端的逻辑通信，但网络层是为主机之间提供逻辑通信（面向主机，承担路由功能，即主机寻址及有效的分组交换）。 各种应用进程之间通信需要“可靠或尽力而为”的两类服务质量，必须由运输层以复用和分用的形式加载到网络层。

  1. 公钥密码体制使用不同的加密密钥和解密密钥。

    公开密钥密码体制的产生主要是因为两个方面的原因：

    • 常规密钥密码体制的密钥分配问题
    • 对数字签名的需求。

    RSA加密算法：

    每个用户有两个密钥：加密密钥PK ={e, n} 和解密密钥 SK ={d, n}。N 为两个大素数 p和 q 之积（素数 p和 q 一般为 100位以上的十进数），e和 d 满足一定的关系。当敌手已知 e和 n 时并不能求出 d。

    设选择了两个素数，p = 7, q = 17。

    计算出 n = pq = 717 = 119。 计算出 f(n) = (p - 1)(q* - 1) = 96。

    从[0, 95]中选择一个与 96 互素的数e。 选 e = 5。然后根据(9-10)式：5d = 1 mod 96

    解出 d。不难得出，d = 77, 因为 ed = 5 77 = 385 = 4 96 + 1 = 1 mod 96。

    于是，公开密钥 PK = (e, n) = {5, 119}, 秘密密钥 SK = {77, 119}。

  2. 网络防病毒技术主要内容是什么？

    网络防病毒技术包括预防病毒、检测病毒和消除病毒等3种技术： 网络防病毒技术的具体实现方法包括对网络服务器中的文件进行频繁地扫描和监测。工作站上采用防病毒芯片和对网络目录及文件设置访问权限等。防病毒必须从网络整体考虑，从方便管理人员的工作着手，通过网络环境管理网络上的所有机器，例如，利用网络唤醒功能，在夜间对全网的客户机进行扫描，检查病毒情况。利用在线报警功能，网络上每一台机器出现故障、病毒侵入时，网络管理人员都能及时知道，从而从管理中心处予以解决。

  3. 防火墙

    防火墙（firewall）作为一种访问控制技术，通过严格控制进出网络边界的分组，禁止任何不必要的通信，从而减少潜在入侵的发生，尽可能降低这类安全威胁所带来的安全风险。

    防火墙的两种技术：

    • 分组过滤路由器
    • 应用网关

    第二层防线是入侵检测系统IDS，它通过对进入网络的分组进行深度分析与检测发现疑似入侵行为的网络活动，并进行报警以便进一步采取相应措施。

  4. 两种密码体系

    • 对称密钥密码体制：加密密钥和解码密钥是使用相同的密码体制。
    • 公钥密码体制：使用不同的加密和解密密钥

    任何加密方法的安全性取决于密钥的长度，以及攻破密文所需的计算量，而不是简单地取决于加密的体制（公钥密码体制或传统加密体制）。我们还要指出，公钥密码体制并没有使传统密码体制被弃用，因为目前公钥加密算法的开销较大，在可见的将来还不会放弃传统加密方法。

    名称翻译 & 解释

    OSI: 开放式系统互联（Open System Interconnection Reference Model）简称OSI ，一种开发的互联网模型，包括七层协议栈

    ISO: 国际标准化组织（International Organization for Standardization，ISO）简称ISO，一个专门制定各项标准的国际组织

    URL：统一资源定位符 （Uniform Resource Locator），用来对互联网上的资源进行定位和访问的一串字符串

    MIME: 多用途互联网邮件扩展类型 (Multipurpose Internet Mail Extensions) 是设定某种扩展名的文件用一种应用程序来打开的方式类型，当该扩展名文件被访问的时候，浏览器会自动使用指定应用程序来打开。

    FDDI：光纤分布式数据接口（Fiber Distributed Data Interface）80年代中期发展起来一项局域网技术，它提供的高速数据通信能力要高于当时的以太网（10Mbps）和令牌网（4或16Mbps）的能力

    TCP: 传输控制协议(Transmission Control Protocol )，一种面向连接、可靠的、基于字节序列的传输层协议

    UDP: 用户数据报协议(User Datagram Protocol)，一种无连接的，尽最大努力保证交付的传输层协议

    TCP/IP: TCP/IP协议栈(TCP/IP Protocol Stack)，是一个网络通信模型，可以看做是对OSI七层协议的简化。包括应用层、传输层、网际IP层、网络接口层四大层协议。

    ARP:地址解析协议（Address Resolution Protocol），一种根据IP地址获取到网卡物理地址的协议

    RARP: 反向地址解析协议（Reversed Address Resolution Protocol），和ARP协议作用相反，将网络中的网卡物理地址转化为其对应的主机IP地址。

    IP: 互联网协议（Internet Protocol ），五层网络协议栈中的网络层协议

    FTP: 文件传输协议（File Transfer Protocol），一种基于TCP的协议应用层，是用于在网络上进行文件传输的一套标准协议，使用客户/服务器模式广域网WAN（Wide Area Network）
