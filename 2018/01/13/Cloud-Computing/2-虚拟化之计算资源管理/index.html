虚拟化之计算资源管理

二、虚拟化之计算资源管理

FusionCompute 功能

  • 完成底层硬件资源的虚拟化功能——CNA（Computing Node Agent）组件

  • 完成虚拟化资源的分配、调度、管理功能——VRM（Virtual Resource Manager）组件

FusionCompute架构

VNA：虚拟节点代理，部署在CNA上，实施计算、存储、网络的虚拟化的配置管理。

CNA

对下完成对上通过VNA与VRM通讯

UVP

VNA

VRM

对下与CNA通讯，收集CNA上报的虚拟资源信息、统计信息和状态信息，对上为管理员提供方便的webui的操作界面，方便管理员查看、监控和操作。

  • 一个VRM可以管理最多32个集群
  • 华为的VRM是以Linux操作系统的方式运行，可以依赖于虚拟机和物理服务器的方式运行。
  • VRM主备部署以提高可靠性，互斥的运行在不同的服务器上。
