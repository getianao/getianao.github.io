在VSCode中新建Eclipse Jave项目和使用Junit

1.准备

首先在pc中安装JDK，并在环境变量和vscode的user-setting中添加Java home，在完成此步后通过java -version验证是否安装成功。

在第一次安装时我选择的时开源的OpenJDK，但在之后VSCode中debug时会出现编译环境版本低于运行环境版本错误；在重新安装Oracle JDK1.8：后解决。报错信息如下：

com/vveye/SW16 has been compiled by a more recent version of the Java Runtime (class file version 53.0), this version of the Java Runtime only recognizes class file versions up to 52.0.

vscode中需要安装插件：

  1. Java Extension Pack：其中包含了Java Test Runner，提供了JUnit 4 (v4.8.0+)、JUnit 5 (v5.1.0+)
    、TestNG (v6.8.0+)的支持，我使用时距离插件发布刚6个月，从github上可以发现现在仍存在一些bug。
  2. Eclipse New Java Project：可以在创建java项目时自动创建src、bin、.classpath、.project。

2.创建Eclipse项目

通过vscode命令面板（ctrl+shift+p）创建java空项目，编辑.classpath文件：

  1. kind="lib"指明了引入的外部jar包，这里我引入了junit5的jar包；声明之后需要在项目根目录下新建lib文件夹，并将包复制进去。

    我不会再在Onedrive的文件夹里写代码了，这里vscode无法解找到Onedrive文件夹的jar包

  2. kind="src"指明了源代码的目录，在这个目录下的文件夹将会被解析为java文件所在的package包。<attribute name="test" value="true"/>表示这个目录含有测试用例，不标注此行会导致vscode无法显示测试类的Code Lens。

  3. kind="con" 标志了运行环境
