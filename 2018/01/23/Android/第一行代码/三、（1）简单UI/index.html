3.2 基本布局

3.2.1 线性布局 LinearLayout

  • android:orientation="horizontal"：排列方向属性
  • android:layout_gravity="top"：对齐方向属性；注意当排列方向属性是horizontal时，只有垂直方向上的对齐方向才会生效；排列方向属性是vertical时，只有水平方向上的对齐方向才会生效；
  • android:layout_weight="1"：用比例的方式控制控件的大小。此时控件的高或宽可设为”0dp”。

3.2.2 相对布局 RelativeLayout

通过相对定位的方式为控件找位置。

  • 相对父控件进行定位
    andriod:layout_alignParentBottom="true"
    andriod:layout_alignParentTop="true"
    andriod:layout_alignParentRight="true"
    andriod:layout_alignParentLeft="true"
    andriod:layout_centerInParent="true"
  • 相对控件进行定位
    android:layout_above="@id/buttom"
    android:layout_below="@id/buttom"
    android:layout_toLeftOF="@id/buttom"
    android:layout_toRightOf="@id/buttom"

  • 相对控件对齐
    android:layout_alignLeft="@id/buttom"：控件与另一控件左边缘对齐
    android:layout_alignRight="@id/buttom"：控件与另一控件右边缘对齐
    android:layout_alignTop="@id/buttom"：控件与另一控件上边缘对齐
    android:layout_alignBottom="@id/buttom"：控件与另一控件下边缘对齐

3.2.3 帧布局 FrameLayout

所有控件默认放在左上角；可以用android:layout_gravity 的方法定位。但由于它的定位方法较少，应用场景较少。

3.2.4 百分比布局

在app/built.gradle 的 dependencies闭包中添加：

[code lang=gradle]
compile 'com.android.support:percent:24.2.1'
[/code]

然后修改布局文件

[code lang=xml]

<android.support.percent.PercentFrameLayout
xmlns:app=”http://schemas.android.com/apk/res-auto"
xmlns:android=”http://schemas.android.com/apk/res/android"

android:layout_width="match_parent"
android:layout_height="match_parent"


</android.support.percent.PercentFrameLayout >

[/code]

类似的 PercentRelativeLayout用法与它相似，它既继承了RelativeLayout的特点，又可以用app:layout_widthPercent用百分比来控制控件的宽高。

3.3 自定义控件

控件和布局的继承结构：

  1. 新建一个布局文件title.xml

    2. [code lang=xml]
    <?xml version=”1.0” encoding=”utf-8”?>



    [/code]

    2.在活动的布局文件中引入自定义控件<include layout ="@layout/title"/>，并隐藏标题栏

    [code lang=text]

    super.onCreate(savedInstanceState);
    setContentView(R.layout.first_layout);
    ActionBar actionBar=getSupportActionBar();
    if(actionBar!=null)
    actionBar.hide();
    [/code]

    1. 可是现在感觉这种方法像是叠加了一种布局，并且每次都要重写自定义控件中的事件监听，那么能不能真正的做成常用控件一样可以直接调用的呢？

      2. 新建一个类让它继承原来布局的类，先用super(context,attr); 和 LayoutInflater.from(context).inflate(R.layout.title,this); 进行初始化，下面进行事件的监听。

      [code lang=java]
      package com.example.getia.activitytest;
      import android.app.Activity;
      import android.content.Context;
      import android.util.AttributeSet;
      import android.view.LayoutInflater;
      import android.view.View;
      import android.widget.Button;
      import android.widget.LinearLayout;
      import android.widget.Toast;

      public class TitleLayout extends LinearLayout
      {
      public TitleLayout(Context context, AttributeSet attr)
      {
      super(context,attr);
      LayoutInflater.from(context).inflate(R.layout.title,this);
      Button titleBack=(Button)findViewById(R.id.title_back);
      Button titleEdit=(Button)findViewById(R.id.title_edit);
      titleBack.setOnClickListener(new OnClickListener() {
      @Override
      public void onClick(View v) {
      ((Activity)getContext()).finish();
      }
      });
      titleEdit.setOnClickListener(new OnClickListener() {
      @Override
      public void onClick(View v) {
      Toast.makeText(getContext(),”You clicked Edit”,Toast.LENGTH_SHORT).show();
      }
      });
      }
      }
      [/code]

      删掉之前的引入布局<include layout ="@layout/title"/>
      最后在活动布局中使用时，就像在使用一般控件一样了，只不过要指明控件的完整类名。

      [code lang=xml]

      <com.example.getia.activitytest.TitleLayout
      android:layout_height=”wrap_content”
      android:layout_width=”match_parent”
      />
      [/code]
