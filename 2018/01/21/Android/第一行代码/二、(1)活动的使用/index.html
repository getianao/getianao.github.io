Android活动的使用

2.1活动的使用

2.1.1 什么是活动

活动（activity）是一种用于和用户进行交互、包含用户界面的组件，一个应用程序可以包含零或多个活动。

2.1.2 活动的创建

创建一个活动包括在java中创建活动以及为活动创建布局。但是这些IDE都可以帮你做好，选择创建一个空活动，可以选择生成布局、设置为主活动、启用向下兼容模式，生成为活动如下：

package com.example.getia.activitytest;

import android.support.v7.app.AppCompatActivity;
import android.os.Bundle;

public class ThirdActivity extends AppCompatActivity {

    @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.third_layout);
    }
}

可以看见活动继承于AppCompatActivity，查看他的声明

Base class for activities that use the support library action bar features.
You can add an to your activity when running on API level 7 or higherby extending this class for your activity and setting the activity theme to or a similar theme.

他事实上是个向前兼容的类。
系统还自动为我们重写了onCreate方法，里面执行了父类的构造函数以及设置了布局。

生成的布局如下：

<?xml version="1.0" encoding="utf-8"?>
<RelativeLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:id="@+id/third_layout"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:paddingBottom="@dimen/activity_vertical_margin"
    android:paddingLeft="@dimen/activity_horizontal_margin"
    android:paddingRight="@dimen/activity_horizontal_margin"
    android:paddingTop="@dimen/activity_vertical_margin"
    tools:context="com.example.getia.activitytest.ThirdActivity">
</RelativeLayout>

我们在里面添加一个按钮：

<?xml version="1.0" encoding="utf-8"?>
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
    android:orientation="vertical"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:weightSum="1">
<Button
    android:id="@+id/button_1"
    android:layout_width="match_parent"
    android:layout_height="wrap_content"
    android:text="Button1" />
</LinearLayout>

在定义一个ID时，方法与引用资源类似，只是多了一个+号：@+id/button_1

再来看看AndroidManifest，如果要设置为主活动，配置如下：

<activity
           android:name=".FirstActivity"
           android:label="This is FirstActivity">
           <intent-filter>
               <action android:name="android.intent.action.MAIN" />
               <category android:name="android.intent.category.LAUNCHER" />
           </intent-filter>
       </activity>

name里的.FirstActivity实际上就是com.example.getia.activitytest.FirstActivity的缩写。label用来设置应用标题以及启动器中的应用名称；下面的 <intent-filter>里的内容作用是注册为主活动

<intent-filter>
                <action android:name="com.example.activitytest.ACTION_START" />
                <category android:name="android.intent.category.MY_CATEGORY" />
            </intent-filter>

2.1.3 多个活动

  1. 建立多个活动与建立活动的方法相似，只是不要选择launch Activity;

  2. 调用

    • 显示intent调用：
      为主活动中的Button关联监听，使用setOnClickListener方法，并重写事件的方法：

      Button button1=(Button)findViewById(R.id.button_1);

      button1.setOnClickListener(new View.OnClickListener(){
          @Override
          public void onClick(View v) {
              Intent intent=new Intent(FirstActivity.this,SencondActivity.class);
         startActivity(intent);
          }
      });

    findViewById() 是通过布局文件中的id来找到布局中按钮，它返回的是一个view对象，需要向下转型；
    intent的构造函数中第一个参数表示当前活动，第二个参数是一个类，表示目标活动；再运用startActivity() 方法将intent传入。
    用这样的方式来其启动活动，称为显示intent。

    • 隐式intent调用

      <intent-filter>
    <action android:name="com.example.activitytest.ACTION_START" />
    <category android:name="android.intent.category.MY_CATEGORY" />
</intent-filter>

    action 标签中表明了这个活动能被com.example.activitytest.ACTION_START 这个action响应；而category 标签中包含了一些附加信息,并且一个活动可以添加多个category。
    在重写事件的方法中使用intent的另一个构造函数public Intent(String action)，这里系统会自动为intent添加默认的category：android.intent.category.MY_CATEGORY 如果要添加自己的category，则调用
    intent.addCategory("com.example.activitytest.MY_CATEGORY")

    然后用 startActivity(intent); 方法让系统在活动中进行匹配。
    这种方法叫做隐式intent

    • 更多隐式intent方法：
      Intent.ACTION_VIEW是Android系统内置的动作

      public void onClick(View v) {

          Intent intent=new Intent(Intent.ACTION_VIEW);
    intent.setData(Uri.parse("http://www.baidu.com"));
    startActivity(intent);
}

    setData 方法可以将一个uri对象作为数据传入intent，这样可以在活动中打开一个网页。

    类似的还有很多内置动作，例如可以打开拨号界面：

    public void onClick(View v) {
               Intent intent=new Intent(Intent.ACTION_DIAL);
               intent.setData(Uri.parse("10086"));
               startActivity(intent);
           }

    2.1.4 在活动中使用Toast

    Toast是在屏幕下方显示一段时间的短小信息；使用方法十分简单，在事件中添加Toast.makeText(this, "You clicked ", Toast.LENGTH_SHORT).show();
    Toast的makeText方法可以创建一个Toast对象，第一个参数就是当前活动，第二个参数是显示的文本内容，第三个参数是显示时长；

    2.1.5 在活动中使用Menu

    首先现在res中建立menu文件夹，再在下建立main文件夹，建立menu文件，添加如下代码：

    <?xml version="1.0" encoding="utf-8"?>
<menu xmlns:android="http://schemas.android.com/apk/res/android">
    <item      android:id="@+id/add_item"
        android:title="Add"
        />
    <item       android:id="@+id/remove_item"
        android:title="Remove"
        />
</menu>

    然后在活动中重写添加菜单的函数

    @Override
public boolean onCreateOptionsMenu(Menu menu) {
    getMenuInflater().inflate(R.menu.main,menu);
    return true;
}

    getMenuInflater() 可以创建一个MenuInflater对象，在调用他的inflate() 方法可以创建菜单，第一个参数为菜单的布局文件，第二个参数是onCreateOptionsMenu的参数。
    返回值为true表示显示菜单，为false表示不显示。

    为了响应菜单中的按键，要重写onOptionsItemSelected() 方法

    public boolean onOptionsItemSelected(MenuItem item)
    {
        switch(item.getItemId()) {
            case R.id.add_item:
                Toast.makeText(this, "You clicked add", Toast.LENGTH_SHORT).show();
                break;
            case R.id.remove_item:
                Toast.makeText(this,"You clicked Remove",Toast.LENGTH_SHORT).show();
                break;
        }
        return true;
    }

    2.1.6 销毁一个活动

    要销毁一个活动有两种方式

    1. 按Back键

    2. finish();

      3. 2.1.7 向下一个活动传递信息

      在用intent创建下一个活动之前，调用
      intent.putExtra("extra_data",data);
      第一个参数是键，第二个是信息，可以是int，String，boolean等等；

      在第二个活动中，按照下面代码可以得到传递的信息；

      Intent intent=getIntent();
String data=intent.getStringExtra("extra_data");

      2.1.8 返回数据给上一个活动

      要能得到生成活动返回的数据，在生成活动上就有所不同，不是 startActivity(intent); 而是 startActivityForResult(intent,1); 第二个参数是请求码，只需要是一个唯一值就行了。

      返回数据到上一个活动是在销毁当前活动时发生的，为当前活动的按钮添加事件：

      public void onClick(View v) {
               Intent intent=new Intent();
               intent.putExtra("data_return","HelloFirstActivity");
               setResult(RESULT_OK,intent);
               finish();
           }

      这里的intent不同于之前创建新活动时的intent，这里只是为了传递数据，我们将数据放到intent中，用setResult()方法返回数据，第一个参数是用来返回处理结果，第二个参数传递参数；
      最后需要结束当前活动。

      然后在之前的活动中重写

      protected void onActivityResult(int requestCode,int resultCode,Intent data)

      方法来得到返回的数据，requestCode是生成活动的唯一请求码，第二个参数是返回的处理结果，第三个参数是含有数据的intent；

      @Override
 protected void onActivityResult(int requestCode,int resultCode,Intent data)
{
switch(requestCode)
case 1:
if(resultCode==RESULT_OK)
{
String returnedData=data.getStringExtra("data_return");
Log.d("FirstActivity",returnedData);
}
 }

      这样在生成活动中按下button之后就会关闭此活动并返回信息给上一活动；

      如果要通过按下Back键关闭活动来返回数据，则要重写onBackPreassed方法；

      @Override
public void onBackPressed()
{
Intent intent=new Intent();
                intent.putExtra("data_return","HelloFirstActivity");
                setResult(RESULT_OK,intent);
}
finish();
