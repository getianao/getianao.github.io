3. The SQL SELECT/FROM/WHERE Framework

  • SELECT/FROM/WHERE语句：

    SELECT 语句指定了查询结果中列出的列。

    FROM 语句指定了查询的表。

    WHERE 语句制定了查询结果中列出的行

    1. The SQL SELECT clause specifies which columns are to be listed in the query results.

    2. The SQL FROM clause specifies which tables are to be used in the query.

    3. The SQL WHERE clause specifies which rows are to be listed in the query results.

  • SQL语句以分号；结尾，尽管一些DBMS允许忽略分号。

    Notice that SQL statements terminate with a semicolon (;) character. The semicolon is required by the SQL standard. Although some DBMS products will allow you to omit the semicolon, some will not, so develop the habit of terminating SQL statements with a semicolon.

  • SELECT语句中指定的列的顺序决定了结果的表中列的顺序。

    The order of the column names in the SELECT phrase determines the order of the columns in the results table.

    In an SQL WHERE clause, if the column contains text or date data, the comparison values must be enclosed in single quotation marks ( ‘{text or date data}’ ). If the column contains numeric data, however, the comparison values need not be in quotes.

6、Performing Calculations in SQL Queries

  • SQL中可以进行特定类型的计算，一类计算是使用 SQL 内置的函数；另一种计算在 SELECT语句中进行简单的算术计算 。

    It is possible to perform certain types of calculations in SQL query statements. One group of calculations involves the use of SQL built-in functions. Another group involves simple arithmetic operations on the columns in the SELECT statement.

  • SQL中内置了五种基本的算术函数，有的DBMS可能在这5种标准的函数上新增一些其他的函数。

    SUM 函数用来计算数目；可以使用 AS 关键字来指定结果中的列名。

    COUNT 函数用来计数；它是唯一一个需要星号的内置函数，表示表中所有的行，它也可以用于任何数据类型，而 SUM, AVG, MIN, MAX 只能用于数字类型的列。当将星号换为列名是，它计算了表中含该列的所有行，但是不会去重，可以使用 DISTINCT 关键字来去重。 但是在 Microsoft Access 中，COUNT 函数中使用 DISTINCT 的用法将会出错，可以使用两重查询来避免。

    There are five SQL built-in functions for performing arithmetic on table columns: SUM, AVG, MIN, MAX, and COUNT.

    COUNT is the only built-in function that requires an asterisk. The COUNT function is also unique because it can be used on any type of data, but the SUM, AVG, MIN, and MAX functions can only be used with numeric data.

9、Querying Two or More Tables with SQL

  • 有两种多表查询的方法：子查询和连接。

    SQL provides two different techniques for querying data from multiple tables: subqueries and joins.

  • 子查询：

  • 连接查询：由于子查询查找的数据只能来自于一张表，要想获得多张表的数据只能使用连接查询。连接查询将两张表合并为一张表，合并后的表的总行数为合并前的表的行数的乘积；所以要加上 WHERE 语句来限制查找的范围。

  • 由两张表连接产生的表叫做连接表；产生这个表的操作叫做连接操作；当使用相等的条件去产生连接表，这种表叫做同等连接；人们说的连接一般都是同等连接；这种类型的连接叫做内连接；

    The table that is formed by concatenating two tables is called a join. The process of creating such a table is called joining the two tables, and the associated operation is called a join operation. When the tables are joined using an equal condition (like the one on OrderNumber), this join is called an equijoin.When people say join, 99.99999 percent of the time they mean an equijoin. This type of join is also referred to as an inner join.

  • 子查询和连接查询的比较

    子查询和连接查询都是多表查询，子查询只能从一张表中获取数据，连接查询可以从多张表中得到信息。所以子查询可以做的连接查询也可以做

    Subqueries and joins both process multiple tables, but they differ slightly. As mentioned earlier, a subquery can only be used to retrieve data from the top table. A join can be used to obtain data from any number of tables. Thus, a join can do everything a subquery can do, and more.
