git分支

1.在使用分支之前

把每一次提交看成一个修改版本，把这些版本按照时间的顺序连接下来形成一个分支，在没有自己创键分支之前，现在的分支叫做主分支，即master分支。master指向主分支中最新的版本；指针HEAD应指向当前分支的指针，所以现在指向master。

每次提交，master分支都会向前移动一步，这样，随着你不断提交，master分支的线也越来越长

下面来看看具体代码操作

#创建dev分支　
git checkout -b dev
#checkout表示切换　-b表示创键，所以等同于以下两行语句

#创键分支
git branch dev
#切换分支
git checkout dev

#查看当前分支
git branch

#如果要将该分支提交到github
git push -u origin dev

#dev分支的工作成果合并到当前分支上
git merge dev

#如果要将该分支提交到github
git push -u origin master

#删除dev分支
git branch -d dev

#删除github上的分支
git push origin :develop-deskmark
