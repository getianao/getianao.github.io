数据结构

本文介绍了软考中有关数据结构的知识点，包括：十字链表、图、堆。

1. 十字链表

当有向图是稀疏的时候，用十字链表来存储有向图可以达到高效的存储效果。

  • 数据结构：

    • 弧：

      tailVex 存储该弧的弧尾对应的顶点。

      headVex 存储该弧的弧头对应的顶点。

      hHink 存储指向该弧的弧头的另一条弧。

      tLink 存储指向该弧的弧尾的另一条弧。

    • 顶点：

      data 存储定点数据。

      firstIn 存储有向图中一条以该节点为弧头的弧。

      firstOut 存储有向图中一条以该节点为弧尾的弧。

  • 一个有向图维护以下一个顶点的链表，称之为十字链表

    顶点列表 xList 的每一个节点为有向图中的一个结点，每条有相同弧头或弧尾的弧组成一个链表，链表的头节点存放在对应结点的xList节点中。如下例：

2.1 最小生成树算法

最小生成树问题：构造连通图的最小代价生成树。

  • Prim（普里姆）算法：时间复杂度是O(n^2^)，与边数无关，适合求边稠密的网中的最小生成树。

    $N=(V,\left{E\right})$是连通网，$TE$是对应最小生成树上的边的集合，$U​$是一个结点的集合。

    初始状态下TE=Ø，U={u~0~}；

    依次重复以下步骤直到U=V（或者TE中有n-1条边）时，$T=(V,\left{TE\right})$为N的最小生成树：寻找这样一条代价最小的边(u,v)，满足u∈U，v∈V-U，然后将这条边添加到TE中，将v添加到U中。

    用自己的话说就是：寻找一端是已选结点、另一端不是且代价最小的边。

  • Kruskal（克鲁斯卡尔）算法：时间复杂度是O(elog~2~e)（e为边的数目），与顶点数无关，适合求边稀疏的网的最小生成树。

    $N=(V,\left{E\right})$是连通网，$TE$是对应最小生成树上的边的集合。

    初始状态下TE=Ø，树$T=(V,\left{TE\right})​$；

    依次重复以下步骤直到TE中有n-1条边时，$T=(V,\left{TE\right})​$为N的最小生成树：寻找这样一条代价最小的边(u,v)，满足这条边落在树T中的不同连通分量上（若不满足舍去这条边，继续寻找），然后将这条边添加到TE中。

    用自己的话说就是：寻找不会构成回路的代价最小的边。

2.2 拓扑排序算法

拓扑排序是由一个集合上的偏序得到一个集合上的全序。偏序是集合仅有部分成员可比较，全序时集合内所有的成员均可比较。

  • 算法过程：

    1. 在AOV中选择一个没有前驱的节点（即入度为0），输出他。
    2. 在图中删除这个节点以及和这个节点有关的弧
    3. 重复以上步骤直至没有入度为0的节点。

2.3 关键路径

完成工程最短的时间是从开始点到完成点的最长路径的长度，这条路径被称为关键路径。（根据这条性质，在AOE网不太复杂时，可以直接把最长路径看出来）

如果$e(i)==l(i)​$，那么活动a(i)为关键活动；关键路径上的活动均为关键活动；

提前完成非关键活动并不能加快完成时间。

符号 含义 计算方法 计算解释
e(i) 活动a(i)的最早开始时间 $e(i)=ve(j)$ 活动i由顶点j出发，活动i的最早开始时间为顶点j的最早发生时间
l(i) 活动a(i)的最迟开始时间（指的是在不推迟整个工程完成的情况下） $l(i)=vl(k)-dut(<j,k>)$ 活动i由状态j指向状态k，活动i的最迟开始时间为顶点k的最迟发生时间减活动i的时间
l(i)-e(i) 完成活动a(i)的活动余量
ve(j) v(j)顶点的最早发生时间 从ve(0)=0向后递推：$ve(j)=Max\left{ve(i)+dut(<i,j>))\right}$
vl(j) v(j)顶点的最迟发生时间 从vl(n-1)=ve(n-1)向后递推：$vl(i)=Min\left{vl(j)-dut(<i,j>))\right}$
dut(<j,k>) 活动a(i)的持续时间

  • 求解关键路径的算法：

    1. 输入e条弧<j，k>，建立AOE-网的存储结构；
    2. 从源点e出发，令ve[0]=0，按拓扑有序求其余各顶点的最早发生时间ve[i]（1≤i≤n-1）。如果得到的拓扑有序序列中顶点个数小于网中顶点数n，则说明网中存在环，不能求关键路径，算法终止；否则执行步骤3；
    3. 从汇点vn出发，令vn[n-1]=ve[n-1]，按逆拓扑有序求其余各顶点的最迟发生时间vl[i]（n-2≥i≥2）；
    4. 根据各顶点的ve和vl值，求每条弧s的最早开始时间e(s)和最迟开始时间l(s)；若某条弧满足条件e(s)=l(s)，则为关键活动。

    例子：

    | 顶点 | ve（第一步，从上到下） | vl（第二步，从下到上） | 活动 | e（第三步） | l（第四步） | l-e（第五步） |
    | —- | ———————- | ———————- | —- | ———– | ———– | ————- |
    | v1 | 0 | | a1 | 0 | 1 | 1 |
    | v2 | 3 | 4 | a2 | 0 | 0 | 0 |
    | v3 | 2 | 2 | a3 | 3 | 4 | 1 |
    | v4 | 6 | 6 | a4 | 3 | 4 | 1 |
    | v5 | 6 | 7 | a5 | 2 | 2 | 0 |
    | v6 | 8 | 8 | a6 | 2 | 5 | 3 |
    | | | | a7 | 6 | 6 | 0 |
    | | | | a8 | 6 | 7 | 1 |

3 堆排序

3.1 堆的定义

堆是具有以下性质的完全二叉树：每个结点的值都大于或等于其左右孩子结点的值，称为大顶堆；或者每个结点的值都小于或等于其左右孩子结点的值，称为小顶堆；并按顺序进行编号，如下图：

该数组从逻辑上讲就是一个堆结构，我们用简单的公式来描述一下堆的定义就是：

大顶堆：arr[i] >= arr[2i+1] && arr[i] >= arr[2i+2]

小顶堆：arr[i] <= arr[2i+1] && arr[i] <= arr[2i+2]

3.2 堆排序的基本思想

  1. 步骤一 构造初始堆。将给定无序序列构造成一个大顶堆（一般升序采用大顶堆，降序采用小顶堆)。
    • 按照给出的无序序列构造初始堆；
    • 从最后一个非叶子结点开始，从左至右，从下至上进行调整：如果父节点小于子节点，则将父节点和子节点交换。
  2. 将堆顶元素与末尾元素进行交换，使末尾元素最大。然后继续调整堆，再将堆顶元素与末尾元素交换，得到第二大元素。如此反复进行交换、重建、交换。

综上所述：堆排序的思想其实就是通过不断的构造大顶堆来找到当前最大的元素。
