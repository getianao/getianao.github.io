github入门

1.新建远程仓库

如果你想把本地的仓库同步到远程，应该完成这部操作。
首先在github网站上新建一个仓库：

Add a license为你添加了一个开源许可证，具体区别可以见下图：

如果本地已经建立好了仓库，在本地仓库上右键选择git bash here

2.建立本地仓库

如果你还没有建立本地仓库，应该先cd到目标路径下，执行下面的操作：

git init ##创建新的 git 仓库

这样本地仓库就建立好了。现在我们就可以实现本地仓库上的版本控制了。
下面我们提交一个版本为例：

git add README.md ##    添加文件到暂存区
##git add -A 所有文件
git commit -m "first commit" ##提交修改信息

只有两步，我们就发布了仓库的新版本。

3.同步远程仓库：

首先cd到本地仓库的目录，输入如下语句：

git remote add origin https://github.com/getianao/test.git

这样我们就把远程仓库关联到本地仓库

然后就可以推送了：

git push -u origin master

这里是推送master分支
此时远程仓库还是空的，可以把本地库的所有内容推送到远程库上，在第一次推送master分支时，加上了-u参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。以后就可以不加-u了。

4.同步仓库到本地

当其他人在远程仓库中做出了更新，或者想下载一个别人的仓库时，可以直接把仓库克隆过来

 git clone https://github.com/getianao/test.git

## 4.工作流
你的本地仓库由 git 维护的三棵"树"组成。第一个是你的 工作目录，它持有实际文件；第二个是 暂存区（Index），它像个缓存区域，临时保存你的改动；最后是 HEAD，它指向你最后一次提交的结果。
所以建立新仓库步骤可以概括如下：

> ①github建立公共仓库
②转到本地仓库路径，初始化本地仓库 *//git init*
③向本地仓库添加文件
④提交文件到暂存区 *//git add **
⑤提交修改信息 *//git commit -m "first commit" *
⑥将仓库连接到远程服务器 *//git remote add origin https://github.com/getianao/test.git *
⑦将这些改动提交到远端仓库 *//git push -u origin master*

## 5.分支
分支是用来将特性开发绝缘开来的。在你创建仓库的时候，master 是"默认的"分支。在其他分支上进行开发，完成后再将它们合并到主分支上。

创建一个叫做"feature_x"的分支，并切换过去：

```bash
 git checkout -b feature_x

切换回主分支：

git checkout master

再把新建的分支删掉：

git branch -d feature_x

除非你将分支推送到远端仓库，不然该分支就是 不为他人所见的：

git push origin <branch>

分支这里之后再详细说明 。
