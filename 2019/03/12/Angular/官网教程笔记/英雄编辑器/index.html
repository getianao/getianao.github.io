组件（component）是angular的基本构造块。整个项目在一个叫做AppComponent的组件下管理运行，并可以继续往这个组件中添加子组件。

AppComponent组件在 src/app 目录下，AppComponent组件主要由下面3个文件构成，其他组件也是如此：

  1. app.component.ts— 组件的类代码，这是用 TypeScript 写的。

  2. app.component.html— 组件的模板，即页面的显示，这是用 HTML 写的。

  3. app.component.css— 组件的私有 CSS 样式。

2.生成组件的ts文件分析

  • 第一部分：类的导入

    从'@angular/core'核心库导入了Component和OnInit，分别是作为装饰器和生命周期钩子。

  • 第二部分：装饰器函数

    @Component装饰器函数为Angular指定了元数据，包括：

    selector：CSS 元素选择器，指定了在父组件html文件中匹配组件的名称，使用<app-heroes></app-heroes>即可引入这个组件；

    templateUrl：组件模板文件的位置；

    styleUrls：组件私有 CSS 样式表文件的位置；

    所谓元数据，就是帮助angular知道需要哪些文件和库，以及如何将整个应用程序组合起来；部分元数据位于@Component装饰器中，还有一部分元数据位于 @NgModule 装饰器中，但 @NgModule 装饰器位于父组件目录下的AppModule类定义文件中。

  • 第三部分：导出类的定义

    定义了组件类，将它声明为导出类是因为需要在其他可能的地方使用这个类，将它继承于OnInit则是为了使用生命周期钩子的初始化函数。
